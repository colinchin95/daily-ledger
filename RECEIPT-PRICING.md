# Centsei — 收據辨識:用量與收費模型設計

## 背景與核心問題

- App 的**所有功能都幾乎零成本**(本機儲存、同步用 Cloudflare KV、報表/預算/備份),唯一有「每次使用都付錢」成本的是 **收據辨識**(呼叫 Claude API)。
- 因此:**收費/限制只該綁在「收據辨識」這一個功能上**;其餘永遠免費 → 衝下載量。
- 成本基準(Haiku 4.5):每掃一張收據 ≈ **US$0.0022 ≈ RM0.01**(約一仙)。100 張 ≈ RM1。

## 🚨 現況風險(上線前必須先解決)

目前 `/receipt` 端點是**公開、無限制**的,且用**開發者你自己的一把 API key**。
若直接給全世界用 = **任何人都能無限呼叫、全部記在你帳上**。
→ 上線前一定要先做「用量上限 + 防濫用」。這不是選配。

## 沒有帳號怎麼計量?

App 主打隱私、無帳號。做法:首次開啟產生一組**匿名安裝 ID**(localStorage 的 UUID),
每次掃收據附帶這個 ID;Worker 在 KV 記**每月計數**並擋下超額。
- 優點:不需 email/密碼,維持隱私定位。
- 限制:清資料/重裝會重置、ID 可被偽造 → 所以**免費額度要設低**,並加防濫用(見下)。

## 建議方案(三層)

| 方案 | 收據辨識 | 成本由誰付 | 價格 |
|---|---|---|---|
| **Free**(預設) | 每月 **5 張** | 你(很低) | 免費 |
| **BYO Key**(自備金鑰) | **無限** | 使用者自己的 Anthropic key | 免費 |
| **Pro**(訂閱) | 無限(防濫用硬上限 500/月) | 你 | ~US$2.99/月 或 US$19.99/年 |

**為什麼這樣分:**
- **Free 5 張**:讓人試用、體驗到「拍照即記帳」的爽點,又把你的成本壓到每人每月 ≤ RM0.05。
- **BYO Key**:給技術型/重度使用者一條**零成本(對你)的無限通道**——他們貼自己的 key,你完全不付費。這也最符合隱私客群。**強烈建議一定要有這層。**
- **Pro 訂閱**:一般人不想搞 key,就付費解鎖無限。利潤極高:就算每月掃 100 張成本才 RM1,訂閱 US$2.99 扣掉金流/抽成仍大賺。

## 防濫用(因為端點公開 + 接的是付費 API)

1. **每月計數上限**(免費 5、Pro 500),Worker 端 KV 強制。
2. **頻率限制**:同一 ID 每小時最多 ~20 張。
3. **Cloudflare Turnstile**:在 `/receipt` 加一道隱形人機驗證,擋掉腳本/機器人刷量(有現成做法)。
4. **圖片大小上限**(已有,前端壓縮 + Worker 擋過大 payload)。

## 規模成本提醒

免費層的 AI 成本**會隨用戶數線性成長**:10 萬活躍用戶 × 每月 5 張 ≈ RM5,000/月。
所以:免費額度保持低、把重度使用者導向 **BYO Key(零成本)** 或 **Pro(有收入)**,
才不會「越紅越虧」。

## 金流怎麼接(跟發行方式綁)

- **網頁/PWA**:用 **Stripe** 訂閱;付款成功後發一張**簽章授權 token**(JWT)存在 App,`/receipt` 驗證後給無限。
- **App Store / Google Play**:用平台**內購(IAP)**,後端驗證收據 → 發放權限(Apple/Google 抽 15–30%)。

## 建議的落地順序(務實版)

- **Phase 0(上線前必做,小工程)**:匿名 ID + 每月計數 + 免費上限 + Turnstile 防濫用。
  → 立刻把「你的成本」封頂,讓收據功能**可以安全地公開給全世界**。**先不做付款。**
- **Phase 1(有人用了)**:加「自備 Anthropic Key」選項(對你零成本,服務重度/隱私用戶)。
- **Phase 2(要賺錢了)**:接 Stripe 訂閱(網頁)或平台 IAP(上架),解鎖 Pro 無限。

> 原則:**有用戶之前不要先蓋金流系統**。先用 Phase 0 安全上線、量測真實用量與成本,再決定定價與是否上架。
